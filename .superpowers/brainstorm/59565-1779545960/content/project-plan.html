4 阶段开发计划

Phase 1: 基础设施 & 协议定义

预计 3-5 天
  • 定义 MQTT 通信协议 (Topic + 消息格式 JSON)
  • 定义 REST API 接口规范 (OpenAPI 3.0)
  • 搭建 PostgreSQL + EMQX Docker 环境
  • Go 项目骨架 + 数据库 Migration
  • ESP32 项目骨架 + 基础 MQTT 连接

Phase 2: 核心功能 — 设备控制链路

预计 5-7 天
  • ESP32: PID温控 + 负离子驱动 + 状态机 + LED
  • Go API: 设备注册/启动/停止/状态查询
  • MQTT 指令下发 + 状态上报闭环
  • 订单创建/计费基础逻辑
  • 管理后台: 设备列表 + 实时状态面板

Phase 3: 高级功能 & 安全加固

预计 4-6 天
  • ESP32: 4G 模块驱动 + Wi-Fi/4G 自动切换
  • ESP32: OTA 固件升级
  • ESP32: 安全看门狗 + 硬保护电路逻辑
  • Go: 批量调参 (按地区/季节)
  • Go: 用户认证 JWT + API 限流
  • Go: 使用报表 + 营收统计
  • 管理后台: 批量管理 + 报表页面

Phase 4: 联调测试 & 上线

预计 3-5 天
  • ESP32-后台-APP 三方联调
  • 压力测试 (多设备并发)
  • 安全测试 (MQTT TLS + API 渗透)
  • 部署文档 + 运维手册
  • 生产环境 Docker 部署

通信协议示例 (MQTT Payload)

// 云端 → 设备: device/{id}/cmd
{"cmd": "start", "tid": "uuid-xxx", "max_heat": 80, "target_out": 35}

// 设备 → 云端: device/{id}/status (每 5s 上报)
{"status": "running", "heat_temp": 72, "out_temp": 34.5, "ion_ok": true, "uptime": 120}

// 设备 → 云端: device/{id}/heartbeat (每 30s)
{"rssi": -45, "heap": 128456, "conn_type": "wifi"}

// 设备 → 云端: device/{id}/event (异常时)
{"event": "over_temp", "value": 85, "limit": 80, "action": "auto_shutdown"}
