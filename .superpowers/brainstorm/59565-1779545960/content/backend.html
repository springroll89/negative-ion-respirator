核心 API 端点

接口 功能 调用方
POST /api/v1/order/create扫码创建订单APP
POST /api/v1/device/start启动设备APP
POST /api/v1/device/stop停止设备APP
GET /api/v1/device/status/:id查询设备状态APP/管理后台
PUT /api/v1/admin/device/config远程调温配置管理后台
GET /api/v1/admin/devices设备列表/批量管理管理后台
POST /api/v1/admin/batch/config按地区/季节批量调参管理后台
GET /api/v1/admin/report/*使用报表/营收统计管理后台

三大核心流程

扫码使用流程
  1. 用户扫码 → APP 调用 /order/create
  2. 后台生成订单 + 验证用户
  3. APP 调用 /device/start
  4. 后台通过 MQTT 下发启动指令
  5. 设备进入 HEATING → RUNNING
  6. 设备定时上报状态
  7. 使用结束 → APP 调 /device/stop
  8. 后台结算订单 + 停止设备
远程温控流程
  1. 管理员在后台修改温度配置
  2. API 校验权限和参数范围
  3. 写入配置到 DB (地区/设备级)
  4. MQTT 下发新配置到目标设备
  5. 设备更新 PID 目标值
  6. 支持按设备/地区/季节批量
